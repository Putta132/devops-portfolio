Production CI/CD Pipeline

  • Architected a multi-stage GitHub Actions CI/CD pipeline for 6 containerized microservices (NovaPay fintech platform) — replacing manual SSH deployments and reducing deployment time from 3 hours to under 8 minutes.
  • Implemented GitOps delivery with ArgoCD watching a dedicated gitops-configs repo; ArgoCD auto-deploys Helm chart changes to AWS EKS within 2 minutes of a merge to main — zero manual kubectl commands required.
  • Integrated Trivy container scanning and SonarQube quality gates into the pipeline — blocking merges on CRITICAL CVEs and enforcing ≥70% code coverage; reduced change failure rate from 40% to under 5%.
  • Implemented Blue/Green deployments using Kubernetes service selectors with k6 smoke tests during traffic switch — achieving zero-downtime releases; rollback executed via a single git revert + push.
  • Configured AWS Secrets Manager with OIDC-based GitHub Actions authentication and External Secrets Operator on Kubernetes — zero hardcoded credentials across entire codebase, verified via git grep.
  • Built Prometheus + Grafana dashboards tracking all four DORA metrics (Deployment Frequency, Lead Time, Change Failure Rate, MTTR) sourced from GitHub Events API and PagerDuty.

Multi-Environment AWS Platform

  • Eliminated ClickOps-related drift incidents by migrating 3 environments (dev/staging/prod) to Terraform IaC, resulting in 100% reproducible infrastructure from a single codebase with zero hardcoded values.
  • Built reusable Terraform modules for VPC (multi-AZ, public/private/database subnets), EKS (managed node groups, OIDC, cluster add-ons), RDS Aurora (multi-AZ, KMS encryption), IAM (least-privilege, no wildcard permissions), and ALB/Route53.
  • Configured S3 + DynamoDB remote state with separate state keys per environment (blast-radius isolation); demonstrated DynamoDB state lock preventing concurrent applies — concurrent runs blocked with lock error showing holder ID.
  • Integrated tfsec and Checkov policy-as-code scanning into GitHub Actions CI on every PR; achieved zero HIGH-severity findings with complete VPC + EKS + RDS Aurora provisioning in under 15 minutes from scratch.
  • Added scheduled terraform plan (every 6 hrs) for automated drift detection with Slack alerts on non-zero exit codes; secrets accessed via AWS Secrets Manager data blocks — never in .tfvars or state in plaintext.
  • Infrastructure outputs (EKS Cluster ARN, ECR Repository URL, RDS Endpoint, VPC Subnet IDs) consumed directly by Project 01’s CI/CD pipeline — full dev/staging/prod environment created with one command: ./scripts/deploy.sh dev.
